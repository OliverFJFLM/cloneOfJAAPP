# day.work クローン (スタティック版)

day.workのコンセプトを再現した、単発・スポットワーク向けマッチングサービスのランディングページです。JavaScriptのみで動作するスタティックサイトとして構成し、検索フォームや案件カード、FAQなどのインタラクションを再現しています。

## 主な特徴

- 📍 **ヒーローセクション**: 実績値・おすすめ案件のスポットライト・週間カレンダーを備えた印象的な導入エリア。
- 🔍 **条件検索フォーム**: キーワード、エリア、カテゴリ、日付、リモート条件でのフィルタリングとクイックタグに対応。
- 📄 **案件リスト**: ダイナミックに生成されるカード表示。残枠メーターやタグ表示、報酬・アクセス情報を掲載。
- 💼 **機能・導入フロー**: ワーカー向け・企業向け双方の利用ステップや特徴を紹介するセクションを実装。
- 💬 **FAQ / 口コミ**: アコーディオンで開閉できるFAQ、利用者の声セクションを用意。
- 📱 **レスポンシブ対応**: 画面幅に合わせてナビゲーション・グリッドが最適化されるようスタイルを調整。

## プロジェクト構成

```
.
├── assets/
│   ├── script.js      # フィルタリングやUI制御を行うフロントエンドロジック
│   └── styles.css     # レイアウトおよびテーマカラーを定義したスタイルシート
├── index.html         # 画面本体。各セクションがHTMLで定義されています
└── vercel.json        # Vercelにデプロイする際の設定（静的ホスティング用）
```

## 使い方

### ローカルでの表示

依存パッケージは不要です。静的サイトとしてそのままブラウザで開けますが、開発時は下記のようにローカルサーバーを立ち上げると便利です。

```bash
python -m http.server 3000
```

上記コマンド実行後、`http://localhost:3000` にアクセスするとページを確認できます。

### デプロイ

Vercelを利用する場合は、このリポジトリをプロジェクトとして指定するだけでデプロイが可能です。`vercel.json` では `@vercel/static` ビルドを使用し、すべてのパスを `index.html` にフォールバックさせる設定としています。

## カスタマイズ方法

- 案件データを更新する場合は `assets/script.js` 内の `jobData` 配列を書き換えてください。
- 既存のセクションを追加・削除したい場合は `index.html` の該当ブロックを編集し、必要に応じて `styles.css` でスタイルを調整してください。
- カラーテーマやフォントなどの全体設定は `styles.css` 冒頭の CSS カスタムプロパティ (`:root`) を変更すると一括で反映されます。
